The package datamanagement is the interface for the storage of the catalogs data. I.e. in order to allow the programm to save to the information the users types in somewhere the programm uses the implementation of the following Classes:
  • {@link net.sourceforge.mecat.catalog.datamanagement.CatalogFactory}
  • {@link net.sourceforge.mecat.catalog.datamanagement.Connection}
  • {@link net.sourceforge.mecat.catalog.datamanagement.Entry}
  • {@link net.sourceforge.mecat.catalog.datamanagement.Catalog}
  • {@link net.sourceforge.mecat.catalog.datamanagement.Identifier}
    •  When implementing the classes you have to consider the following:
    1. You need an implementation of all 5 classes from this package in order to get things work. In most cases this means that you have to implement all 5 classes.
    2. The implementation of the 5 classes should be placed in the same directory. And in the current version this directory has to be a direct subdirectory of this package directory
    3. Last but not least follow the implementation advices from the four class:
    4. {@link net.sourceforge.mecat.catalog.datamanagement.CatalogFactory}
    5. {@link net.sourceforge.mecat.catalog.datamanagement.Connection}
    6. {@link net.sourceforge.mecat.catalog.datamanagement.Entry}
    7. {@link net.sourceforge.mecat.catalog.datamanagement.Catalog}
    8. {@link net.sourceforge.mecat.catalog.datamanagement.Identifier}
